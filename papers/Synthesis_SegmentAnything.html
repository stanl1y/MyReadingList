SECTION 01

One Machine, Four Expansions

TL;DR

這 4 篇是同一台「可提示分割機器」的四次維度擴張:SAM 用空間提示把分割做成 foundation model;SAM 2 加時間(streaming memory → 影片);SAM 3 加語意(概念提示 → 所有實例);SAM 3D 加維度(單圖 → 完整 3D)。介面一直在變,配方從未變:任務可提示化 + 最小架構增量 + model-in-the-loop 資料引擎 + 隨附新 benchmark。

These 4 papers are four dimension-expansions of one promptable segmentation machine: SAM turns segmentation into a foundation model via spatial prompts; SAM 2 adds time (streaming memory → video); SAM 3 adds semantics (concept prompts → every instance); SAM 3D adds dimension (one image → full 3D). The interface keeps changing; the recipe never does: promptable task + minimal architecture delta + a model-in-the-loop data engine + a shipped benchmark.

SECTION 03

The Invariant Recipe

每一代都走同樣四步Every generation takes the same four steps

  1. 重定義任務 — 把新能力寫成「可提示任務」:SA → PVS → PCS → 3Dfy。同一個模型因此既是下游介面,也是資料引擎裡的標註工具。Redefine the task — write the new capability as a "promptable task": SA → PVS → PCS → 3Dfy. The same model is thus both the downstream interface and the annotation tool inside the data engine.
  2. 最小架構增量 — 不重造,只加一個關鍵模組:SAM 2 = SAM + streaming memory;SAM 3 = SAM 2 追蹤器 + DETR 偵測器 + presence token;SAM 3D = flow-matching 幾何/紋理 + MoT layout。Minimal architecture delta — do not rebuild; add one key module: SAM 2 = SAM + streaming memory; SAM 3 = SAM 2 tracker + DETR detector + presence token; SAM 3D = flow-matching geometry/texture + MoT layout.
  3. Model-in-the-loop 資料引擎 — 人只做模型不會的,模型做人嫌慢的:SA-1B(99.1% 全自動)→ SA-V(標註 8.4× 快)→ SA-Co(MLLM AI 驗證器)→ MITL-3DO(驗證代替創作,~80% 自產)。Model-in-the-loop data engine — humans do only what the model cannot; the model does what humans find slow: SA-1B (99.1% automatic) → SA-V (8.4× faster annotation) → SA-Co (MLLM AI verifiers) → MITL-3DO (verify instead of author, ~80% self-made).
  4. 隨附新 benchmark — SA-23 → SA-V → SA-Co(207K 概念)→ SA-3DAO:自己定義賽道。但 provenance 與訓練資料同源(見 05)。Ship the benchmark — SA-23 → SA-V → SA-Co (207K concepts) → SA-3DAO: define your own racetrack. But its provenance is shared with the training data (see 05).

嚴格泛化:新一代「包含」上一代Strict generalization: each generation contains the last

  • SAM 2 空記憶 = SAM(影像被當成單幀影片)——介面完全向後相容,還 6× 快、更準(SA-23 58.9 vs 58.1)。
  • SAM 2 with empty memory = SAM (an image is a one-frame video) — fully backward-compatible, and 6× faster and more accurate (SA-23 58.9 vs 58.1).
  • SAM 3 同時支援 PCS + PVS,且在舊賽道 VOS 上反超 SAM 2(MOSEv2 +6.5)——升級不丟舊能力。
  • SAM 3 supports both PCS + PVS, and beats SAM 2 on the old VOS track (MOSEv2 +6.5) — upgrading loses nothing.
  • SAM 3D 對 mask 來源不可知(SAM/SAM2/SAM3 皆可)且對過/欠分割穩健——家族內部鬆耦合、可自由組裝。
  • SAM 3D is mask-source agnostic (SAM/SAM2/SAM3 all work) and robust to over/under-segmentation — loosely coupled, freely composable within the family.
策略解讀Strategic read

這不是巧合,是產品紀律:每代發布時,舊介面使用者零成本升級;對研究者,「同介面、換引擎」等於天然的控制變因。學術論文很少這樣自我約束——這是把 foundation model 當產品線在經營。[導讀補充]

Not a coincidence but product discipline: at each release, users of the old interface upgrade for free; for researchers, "same interface, new engine" is a natural controlled variable. Academic papers rarely self-constrain like this — the family is run as a product line. [guide note]

最小增量一覽The deltas at a glance
SAM→+ memory→+ detector · presence→+ 3D flow matching
每一步都只加「一個新器官」,其餘沿用——所以每代的增益可以歸因。 Each step grafts exactly one new organ onto the last body — so每 generation gains stay attributable.
⚠ 別誤讀成「越做越大」Not "ever bigger models"
參數其實克制:SAM 3 全模型 ~848M、SAM 3D 1.2B+600M——遠小於同期 LLM。重的是資料引擎與人力迴圈,不是模型。 Parameters stay modest: SAM 3 is ~848M total, SAM 3D 1.2B+600M — tiny next to contemporary LLMs. The heavy investment is the data engine and human loop, not the model.

效率一路坍縮成本;但「模型驗自己的作業」的循環性也一路加深——自動標註若有系統性偏差(邊界風格、物體粒度),飛輪會放大而非清洗它。四代都沒有正面量化這個風險。

Costs collapse — but the circularity of "the model grading its own homework" deepens too: systematic biases in auto-annotation (boundary style, object granularity) get amplified, not cleaned. None of the four papers quantifies this risk head-on.

跨串連結:SAM 3D × VGGTCross-thread: SAM 3D × VGGT 同在你的庫裡的 VGGT 是 3D 的另一條正交路線:VGGT 量測可視幾何(多視角前饋回歸),SAM 3D 生成完整物體(含看不見的背面,靠先驗腦補),甚至可吃 pointmap 當條件——一測一生,互補可組合。 VGGT, also in this library, is the orthogonal 3D route: VGGT measures visible geometry (feed-forward multi-view regression) while SAM 3D generates complete objects (unseen backs included, via priors) — and can even take a pointmap as conditioning. One measures, one imagines; they compose.

SECTION 05

Open Problems Across the Series

整條線共同的坑Pitfalls shared by the whole series

  • 語意天花板:v1 完全 class-agnostic;到 v3 也只到「短名詞片語」——長指涉句與推理要外掛 MLLM(SAM 3 Agent)。語言端始終是「別人的」。
  • The semantics ceiling: v1 is fully class-agnostic; even v3 stops at short noun phrases — long referring expressions and reasoning need an external MLLM (SAM 3 Agent). The language side is always borrowed.
  • 「所有實例」的代價:SAM 2 每物體獨立跑、無跨物體溝通;SAM 3 影片延遲隨物體數線性增長。百物體場景的效率未解。
  • The cost of "every instance": SAM 2 runs per-object with no cross-object communication; SAM 3 video latency grows linearly with object count. Hundred-object scenes remain unsolved.
  • 循環性(見 04):自產標註的系統性錯誤模式可能被放大;benchmark 與訓練資料同源,外部驗證偏弱。
  • Circularity (see 04): systematic error modes in self-annotation may amplify; benchmarks share provenance with training data, external validation is thin.
  • 重現性:配方的核心是 Meta 規模的資料引擎與人力迴圈——學界能微調權重,無法複現方法論本身。
  • Reproducibility: the heart of the recipe is a Meta-scale data engine and human loop — academia can fine-tune the weights but cannot reproduce the methodology.
  • 3D 側:64³ voxel 解析度讓細結構/手/臉變形;一次一物體、無物理一致性(互穿);輸出是先驗樣本,不是量測。
  • On the 3D side: 64³ voxels distort thin structures/hands/faces; one object at a time with no physical consistency (interpenetration); outputs are prior samples, not measurements.
  • 「anything」有邊界:細粒度(機型)、醫療、熱影像等跨域 zero-shot 仍弱(SAM 3 自承)。
  • "Anything" has borders: fine-grained (aircraft types), medical, and thermal cross-domain zero-shot remain weak (SAM 3 admits this).

給讀書會的犀利問題Sharp questions for a reading group

  1. 架構 vs 資料:把四代資料引擎的產出餵給非 SAM 架構(Mask2Former 系、純 DETR),差距還剩多少?功勞從未被拆開。
  2. Architecture vs data: feed the four data engines’ output to a non-SAM architecture (Mask2Former-style, plain DETR) — how much gap remains? The credit has never been split.
  3. 飛輪的錯誤模式:自動標註的偏差(邊界風格、粒度偏好)是否已滲進 SA-1B→SA-V→SA-Co 每一代?能否設計實驗量測?
  4. Flywheel error modes: have auto-annotation biases (boundary style, granularity preference) already seeped through SA-1B→SA-V→SA-Co? Can an experiment measure this?
  5. 語言邊界:PCS 停在短名詞片語是能力限制,還是任務設計選擇?把組合語意外包給 MLLM,等不等於承認「開放詞彙分割」不該是獨立模型?
  6. The language boundary: is stopping at short noun phrases a capability limit or a task-design choice? Does outsourcing compositional semantics to an MLLM concede that open-vocabulary segmentation should not be a standalone model?
  7. 偏好評測的偏誤:SAM 3D 的 5:1 人類偏好由作者方執行,而第三方 ISO3D 上僅打平——哪個更接近真實能力?
  8. Preference-test bias: SAM 3D’s 5:1 human preference was run by the authors, while it merely ties on third-party ISO3D — which is closer to true capability?
  9. 下一個維度:照「空間→時間→語意→3D」的軌跡,下一步是物理/互動嗎?在人類難以驗證的物理量上,「驗證代替創作」的資料引擎還走得通嗎?
  10. The next dimension: after space→time→semantics→3D, is physics/interaction next? Does "verify instead of author" still work for physical quantities humans cannot eyeball?
⚠ 別過度外推Don’t over-extrapolate
這串證明的是「介面 + 資料引擎」路線在感知任務上的威力;它沒有證明這條路能直接通向理解、推理或物理——那些正是它一路外包(MLLM)或迴避(物理一致性)的部分。 The series proves the power of the interface + data engine route for perception; it does not prove the route leads to understanding, reasoning, or physics — precisely the parts it outsources (MLLMs) or sidesteps (physical consistency).
與其他串的對照Against the other threads 力覺串在解「訊號怎麼用」、VGGT 在解「幾何怎麼回歸」,SAM 系列在解「標註怎麼自我複製」——三串合起來剛好是感知系統的三個瓶頸。 The force thread tackles "how to use a signal," VGGT "how to regress geometry," and the SAM series "how annotation replicates itself" — together the three threads map three bottlenecks of perception systems.

可借的配方What to borrow

  • 任務可提示化:把新能力定義成「對任意提示回傳有效輸出」,模型就同時是產品介面與標註工具。
  • Promptable-ize the task: define the capability as "valid output for any prompt," and the model doubles as product interface and annotation tool.
  • 資料引擎優先於架構:設計「人只做驗證」的迴圈,讓自產比例逐代上升——這比換 backbone 值錢。
  • Data engine before architecture: build loops where humans only verify, and let the self-made share climb — worth more than a new backbone.
  • 嚴格泛化紀律:新版本必須「包含」舊介面(空記憶 = SAM 是典範)——使用者零成本升級,研究可歸因。
  • Strict-generalization discipline: each version must contain the old interface (empty memory = SAM is the exemplar) — free upgrades for users, attributable gains for research.
  • 隨附 benchmark,但補外部驗證:定義賽道的同時,主動在第三方基準上報數,並揭露 provenance。
  • Ship the benchmark, add external validation: define the track, but also report on third-party benchmarks and disclose provenance.
